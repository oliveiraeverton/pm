# Commits Semânticos

Os commits semânticos são uma convenção de nomenclatura de mensagens de commit que seguem um padrão específico para comunicar claramente a intenção de cada alteração no código. Eles são úteis para manter um histórico de alterações organizado e compreensível, facilitando a colaboração entre membros da equipe e ajudando na geração automática de notas de versão.

## Padrão de Mensagem

1. **feat**: Para novas funcionalidades ou adições significativas ao código.
2. **fix**: Para correções de bugs ou resoluções de problemas.
3. **chore**: Para tarefas de manutenção, atualizações de dependências, etc.
4. **docs**: Para alterações na documentação, como atualizações de README, comentários de código, etc.
5. **style**: Para melhorias cosméticas, como formatação, identação, etc.
6. **refactor**: Para refatorações de código, sem adição ou remoção de funcionalidades.
7. **test**: Para adição ou modificação de testes.
8. **manutencao**: Para acrescentar funcionalidades.

Exemplo de uma mensagem de commit semântico:

Caso faça a alteração em uma função menu em que acrescenta mais uma opção para o usuário escolher, a mensagem poderia ser:

manutencao(menu): acrescetamos a opção 8- imprimir relatório.
